# `SN-mt/1` — `PermutationError`

Raised when Metamorphose cannot construct a permutation from the
arguments supplied.

## Cause

Permutations in Metamorphose are built from an index sequence over a
positional base. The error variants flag the four ways the indices can
be inconsistent with the requested base or input length. The message
is `could not construct permutation because $reason`.

## Variants

### `SN-mt/1.1` — `BaseRange`

A positional value is greater than or equal to its base.

**Example.** `[↯SN-mt/1.1] could not construct permutation because the value 5 is too large for its positional base 4`

**Resolution.** Either reduce the offending value or increase the
base of the permutation.

### `SN-mt/1.2` — `DuplicateIndex`

The same index appears at two distinct positions, which would map two
inputs to the same output.

**Example.** `[↯SN-mt/1.2] could not construct permutation because the index 3 was duplicated at 5`

**Resolution.** Each index must appear exactly once; deduplicate the
input.

### `SN-mt/1.3` — `InvalidIndex`

An index is outside the valid range `0..max`.

**Example.** `[↯SN-mt/1.3] could not construct permutation because the index appears, but every index should be in the range 0-7`

**Resolution.** Use only indices within the size of the permutation.

### `SN-mt/1.4` — `TooShort`

The input has fewer elements than the permutation requires.

**Example.** `[↯SN-mt/1.4] could not construct permutation because the input, of size 3, is too short for the permutation of size 5`

**Resolution.** Supply an input at least as long as the permutation
size, or build a smaller permutation.

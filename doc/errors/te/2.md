# `SN-te/2` — `ConnectError`

Raised when Telekinesis cannot establish a TCP/TLS connection.

## Cause

`ConnectError` reports a failure during connection setup, before any
HTTP exchange can occur. The message is `the TCP connection failed
because $reason`.

## Variants

### `SN-te/2.1` — `Dns`

The server's hostname could not be resolved.

**Example.** `[↯SN-te/2.1] the TCP connection failed because the server address could not be resolved by DNS`

**Resolution.** Verify the hostname is correct and DNS is reachable.

### `SN-te/2.2` — `Refused`

The server actively refused the connection (e.g. nothing listening
on the port).

**Example.** `[↯SN-te/2.2] the TCP connection failed because the connection was refused`

**Resolution.** Confirm the server is running and listening on the
expected port; check firewalls.

### `SN-te/2.3` — `Ssl`

The TLS handshake failed. The nested `Ssl.Reason` distinguishes
`Handshake`, `Key`, `Peer`, and `Protocol` failures.

**Example.** `[↯SN-te/2.3] the TCP connection failed because the SSL/TLS layer could not be established because the remote peer's identity could not be verified`

**Resolution.** Inspect the inner reason: check certificates and
trust chain (`Peer`), key material (`Key`), supported protocol
versions (`Protocol`), or cipher-suite negotiation (`Handshake`).

### `SN-te/2.4` — `Timeout`

The server did not respond within the allowed time.

**Example.** `[↯SN-te/2.4] the TCP connection failed because the server did not respond within the time limit`

**Resolution.** Retry with a longer timeout, or investigate why the
server is slow.

### `SN-te/2.5` — `Unknown`

The cause does not match any of the more specific variants.

**Example.** `[↯SN-te/2.5] the TCP connection failed because an unrecognized error occurred`

**Resolution.** Inspect logs and any cause attached to the error.

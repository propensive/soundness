# `SN-te/4` — `HttpResponseError`

Raised when Telekinesis cannot parse an HTTP response.

## Cause

After a connection is established, the response is read and parsed.
This error reports a malformed response. The message is `could not
parse HTTP response because $reason`.

## Variants

### `SN-te/4.1` — `Expectation`

A specific character was expected at the parser's current position
but a different one was encountered.

**Example.** `[↯SN-te/4.1] could not parse HTTP response because X was found when : was expected`

**Resolution.** Confirm the server is producing valid HTTP; if the
peer is non-standard, point Telekinesis at a compliant implementation.

### `SN-te/4.2` — `Status`

The HTTP status code on the response line was not three digits or
otherwise unparseable.

**Example.** `[↯SN-te/4.2] could not parse HTTP response because the HTTP status code 12 was invalid`

**Resolution.** The peer is producing a non-conformant status line;
fix the server or use a different one.

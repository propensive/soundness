# `SN-su/1` — `WatchError`

Raised when Surveillance cannot register a path with the operating
system's filesystem watcher.

## Cause

Surveillance wraps the JDK's `WatchService` API. Several distinct
conditions can cause registration to fail; the constructor takes
a `reason: Reason`. The rendered message is `the path could not
be watched because $reason`.

## Variants

### `SN-su/1.1` — `LimitExceeded`

The operating system's per-process or per-user limit on the
number of inotify (or equivalent) watches has been exceeded.

**Example.** `[↯SN-su/1.1] the path could not be watched because the operating system's limit on watched paths has been exceeded`

**Resolution.** Reduce the number of watched paths, or raise the
OS limit (e.g. `fs.inotify.max_user_watches` on Linux).

### `SN-su/1.2` — `Nonexistent`

The path passed to `watch` does not exist.

**Example.** `[↯SN-su/1.2] the path could not be watched because the path does not exist`

**Resolution.** Confirm the path before watching, or create it
first.

### `SN-su/1.3` — `NotDirectory`

The JDK `WatchService` only registers directories; the supplied
path was a regular file or other entry.

**Example.** `[↯SN-su/1.3] the path could not be watched because the path is not a directory`

**Resolution.** Watch the parent directory and filter events on
the file name (Surveillance's `watch` extension does this when
given a file path).

### `SN-su/1.4` — `PermissionDenied`

The current user does not have permission to read the path.

**Example.** `[↯SN-su/1.4] the path could not be watched because the user does not have permission to watch the path`

**Resolution.** Adjust filesystem permissions, or run with a user
that can read the directory.

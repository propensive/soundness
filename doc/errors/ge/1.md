# `SN-ge/1` — `MediaTypeError`

Raised when a value cannot be parsed as a valid media type (a.k.a.
MIME type).

## Cause

Gesticulate parses media types of the form `type/subtype[+suffix]
[; parameter=value]*`. The variants flag specific syntactic problems.
The message is `the value $value is not a valid media type;
${reason.message}`.

## Variants

### `SN-ge/1.1` — `NotOneSlash`

The value did not contain exactly one `/` between type and subtype.

**Example.** `[↯SN-ge/1.1] the value text/html/extra is not a valid media type; a media type should always contain exactly one '/' character`

**Resolution.** Provide one `/` between top-level type and subtype.

### `SN-ge/1.2` — `MissingParam`

The value ends with `;`, suggesting a parameter that was not
written.

**Example.** `[↯SN-ge/1.2] the value text/html; is not a valid media type; a terminal ';' suggests that a parameter is missing`

**Resolution.** Either add the parameter or remove the trailing `;`.

### `SN-ge/1.3` — `InvalidGroup`

The top-level type is not one of the registered groups (e.g.
`text`, `image`, `application`, …).

**Example.** `[↯SN-ge/1.3] the value foo/bar is not a valid media type; the type must be one of: ...`

**Resolution.** Choose a registered top-level type.

### `SN-ge/1.4` — `InvalidChar`

A character that is not allowed in media-type tokens appears.

**Example.** `[↯SN-ge/1.4] the value text/ht ml is not a valid media type; the character ' ' is not allowed`

**Resolution.** Remove the offending character; tokens cannot
contain whitespace or special characters.

### `SN-ge/1.5` — `InvalidSuffix`

The structured-syntax suffix (after `+`) is not recognised.

**Example.** `[↯SN-ge/1.5] the value application/foo+xyz is not a valid media type; the suffix '+xyz' is not recognized`

**Resolution.** Use a documented suffix such as `+xml`, `+json`, or
omit the suffix.

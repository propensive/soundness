# `SN-ge/2` — `MultipartError`

Raised when multipart MIME data cannot be parsed.

## Cause

Gesticulate parses multipart bodies bounded by a delimiter declared
in the `Content-Type` header. Each part has its own headers and
body. This error reports a structural problem with the multipart
stream. The message is `multipart data could not be read because
$reason`.

## Variants

### `SN-ge/2.1` — `Expected`

A specific character was expected at the parser's position but a
different one was encountered.

**Example.** `[↯SN-ge/2.1] multipart data could not be read because the character ':' was expected`

**Resolution.** Confirm the producer is emitting valid multipart
data; check for missing CRLFs or boundaries.

### `SN-ge/2.2` — `StreamContinues`

Bytes appeared after the closing boundary.

**Example.** `[↯SN-ge/2.2] multipart data could not be read because the stream continues beyond the last part`

**Resolution.** Truncate the body at the closing boundary, or stop
emitting bytes after it.

### `SN-ge/2.3` — `BadBoundaryEnding`

Content followed a boundary that does not match the documented
terminator format.

**Example.** `[↯SN-ge/2.3] multipart data could not be read because unexpected content followed the boundary`

**Resolution.** Boundaries must be followed either by `\r\n` (start
of next part) or `--\r\n` (end of body); fix the producer.

### `SN-ge/2.4` — `MediaType`

A part's `Content-Type` header is not a valid media type.

**Example.** `[↯SN-ge/2.4] multipart data could not be read because the media type is invalid`

**Resolution.** See `SN-ge/1` for media-type rules.

### `SN-ge/2.5` — `BadDisposition`

A part's `Content-Disposition` header has the wrong format.

**Example.** `[↯SN-ge/2.5] multipart data could not be read because the `Content-Disposition` header has the wrong format`

**Resolution.** Use the standard form, e.g.
`form-data; name="field"; filename="x.txt"`.

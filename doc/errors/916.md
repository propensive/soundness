# `SN-916` — `PojoError`

Raised when Austronesian cannot deserialize a value from its
classloader-neutral "Pojo" representation.

## Cause

Austronesian encodes Scala values into a representation built only
from JDK-standard types — boxed primitives, `String`, and
`Array[Object]` — so they can cross classloader or process
boundaries. This error is raised during decoding, when a Pojo value
does not match the shape the target type expects: a scalar where an
array of fields was expected, an array of the wrong length, or a
sum value that is not a `(label, payload)` pair.

## Example message

```
[↯SN-916] could not deserialize from Java POJO types
```

## Resolution

Confirm that the value was encoded from the same type it is being
decoded to, and that both sides agree on the type's definition —
field order and variant names are part of the encoding. More
specific reasons may be added as the module evolves.

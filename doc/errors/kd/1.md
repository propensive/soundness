# `SN-kd/1` ‚Äî `RegexError`

Raised when Kaleidoscope cannot parse a regular expression literal.
Each variant pinpoints the syntactic problem.

## Cause

The `r"‚Ä¶"` regex interpolator parses the pattern at compile time. When
the parser cannot accept the input, it throws a `RegexError` carrying
the character `index` at which the problem was detected and a `Reason`
explaining what was expected. The constructed message is `the regular
expression could not be parsed because $reason at $index`.

## Variants

### `SN-kd/1.1` ‚Äî `UnclosedGroup`

A capturing group was opened with `(` but never closed with `)`.

**Example.** `[‚ÜØSN-kd/1.1] the regular expression could not be parsed because a capturing group was not closed at 11`

**Resolution.** Add the missing `)` or escape the `(` if a literal
parenthesis is intended.

### `SN-kd/1.2` ‚Äî `ExpectedGroup`

An extractor pattern was used in a context that requires a capturing
group to follow it, but no group was found.

**Example.** `[‚ÜØSN-kd/1.2] the regular expression could not be parsed because a capturing group was expected immediately following an extractor at 0`

**Resolution.** Wrap the extracted portion of the pattern in `(‚Ä¶)`.

### `SN-kd/1.3` ‚Äî `BadRepetition`

A repetition range `{n,m}` has `m < n`.

**Example.** `[‚ÜØSN-kd/1.3] the regular expression could not be parsed because the maximum number of repetitions is less than the minimum at 5`

**Resolution.** Swap the bounds, or use `{n,}` if no upper bound is
needed.

### `SN-kd/1.4` ‚Äî `Uncapturable`

A capturing group is nested inside a repeating group, which means it
cannot be extracted unambiguously.

**Example.** `[‚ÜØSN-kd/1.4] the regular expression could not be parsed because a capturing group inside a repeating group can not be extracted at 7`

**Resolution.** Either remove the capture by switching to a
non-capturing group `(?:‚Ä¶)`, or restructure the pattern so the
repetition is inside the capture rather than outside it.

### `SN-kd/1.5` ‚Äî `UnexpectedChar`

A character that does not belong in a repetition range was
encountered.

**Example.** `[‚ÜØSN-kd/1.5] the regular expression could not be parsed because the repetition range contained an unexpected character at 9`

**Resolution.** Use only digits and a single comma between `{` and
`}`.

### `SN-kd/1.6` ‚Äî `NotInGroup`

A `)` was encountered with no corresponding `(`.

**Example.** `[‚ÜØSN-kd/1.6] the regular expression could not be parsed because a closing parenthesis was found without a corresponding opening parenthesis at 4`

**Resolution.** Add the missing `(`, escape the `)` if it should be
literal, or remove the stray closing parenthesis.

### `SN-kd/1.7` ‚Äî `IncompleteRepetition`

A repetition range was opened with `{` but never closed with `}`.

**Example.** `[‚ÜØSN-kd/1.7] the regular expression could not be parsed because the repetition range was not closed at 8`

**Resolution.** Add the missing `}`.

### `SN-kd/1.8` ‚Äî `InvalidPattern`

The pattern was rejected by the underlying parser for a reason that
does not fit any of the more specific variants.

**Example.** `[‚ÜØSN-kd/1.8] the regular expression could not be parsed because the pattern was invalid at 3`

**Resolution.** Review the pattern around the reported index; consult
the regex syntax for the construct in use.

### `SN-kd/1.9` ‚Äî `UnclosedEscape`

The pattern ends with a `\` that has nothing to escape.

**Example.** `[‚ÜØSN-kd/1.9] the regular expression could not be parsed because nothing followed the escape character `\` at 12`

**Resolution.** Either remove the trailing backslash or escape the
character that should follow it.

### `SN-kd/1.10` ‚Äî `EmptyCharClass`

A character class `[‚Ä¶]` contained no characters.

**Example.** `[‚ÜØSN-kd/1.10] the regular expression could not be parsed because the character class is empty at 2`

**Resolution.** Add at least one character to the class, or remove
the empty `[]`.

### `SN-kd/1.11` ‚Äî `ZeroMaximum`

A repetition range had a maximum of zero, e.g. `{0,0}` or `{0}`.

**Example.** `[‚ÜØSN-kd/1.11] the regular expression could not be parsed because the maximum number of repetitions must be greater than zero at 6`

**Resolution.** Use a maximum greater than zero, or remove the
repeated subpattern entirely if it should never match.

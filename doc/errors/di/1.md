# `SN-di/1` — `DiffError`

Raised when Dissonance cannot parse a unified-diff input.

## Cause

Dissonance's diff parser walks the input line by line. When a line
does not match any expected diff syntax (header, hunk header, `+`
addition, `-` removal, or context line) at the position where it
appears, this error is thrown with the line number and the
offending text.

## Example message

```
[↯SN-di/1] could not read the diff at line 14: ??? unexpected
```

## Resolution

Confirm the input is a well-formed unified diff; check for stray
text or a truncated patch.

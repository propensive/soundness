# `SN-hc/1` — Cannot match against a doctype declaration

Raised at compile time by Honeycomb's HTML pattern matching when a
pattern attempts to match against a `<!DOCTYPE …>` declaration.

## Cause

Honeycomb's HTML extractors recognise element nodes; document-type
declarations carry no extractable structure for pattern matching.

## Example message

```
[↯SN-hc/1] cannot match against a document type declaration
```

## Resolution

Match against an element node, not the doctype.

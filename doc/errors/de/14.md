# `SN-de/14` — Block-comment usage

Raised by the Decorum compiler plugin when a `/* … */` block comment
appears outside its permitted use.

## Cause

Soundness reserves block comments for the licence header (lines
1–32). All other comments must be line comments (`//`).
Documentation should live in `doc/` markdown rather than `/** … */`
docstrings.

## Variants

### `SN-de/14.1` — No `/** … */` doc comments

A scaladoc-style block comment was found.

**Example.** `[↯SN-de/14.1] `/** ... */` block comments are not permitted; use `doc/` markdown instead`

**Resolution.** Convert the docstring into prose under `doc/`, or
into a line comment if it explains a non-obvious mechanism inline.

### `SN-de/14.2` — No `/* … */` block comments outside the licence

A plain block comment appeared outside lines 1–32.

**Example.** `[↯SN-de/14.2] `/* ... */` block comments are reserved for the license header (lines 1-32)`

**Resolution.** Replace with one or more line comments.

# `SN-de/7` — `package` declaration on line 33

Raised by the Decorum compiler plugin when line 33 is not
`package <module>` matching the source's module.

## Cause

After the 32-line licence header, line 33 must declare the package.
The package name must be a single identifier segment matching the
file's expected module.

## Example messages

```
[↯SN-de/7] expected `package` declaration on line 33, found content on line 34
```

```
[↯SN-de/7] package declaration must be a single identifier segment, not `foo.bar`
```

```
[↯SN-de/7] package `foo` does not match expected module `bar`
```

```
[↯SN-de/7] package declaration must contain only `package <ident>` on line 33
```

```
[↯SN-de/7] line 33 must be `package <module>`
```

## Resolution

Place a single `package <module>` declaration on line 33, where
`<module>` is the module the file belongs to.

# `SN-de/9` — Top-level import rules

Raised by the Decorum compiler plugin when a top-level import
violates the project's import-style rules.

## Cause

Decorum scans top-level imports and enforces three rules: no aliases,
ordering by group, and group separation by blank lines. Violations
are emitted with rule tags `9.1`, `9.2`, and `9.3`.

## Variants

### `SN-de/9.1` — No aliases

Top-level imports must not use `as` or `=>` aliases — write the full
path.

**Example.** `[↯SN-de/9.1] top-level imports must not use aliases (`as` or `=>`); write the full path`

**Resolution.** Remove the alias and refer to the full path
throughout the file.

### `SN-de/9.2` — Group order

Imports must be in the documented group order, and within a group
must appear in alphabetical order.

**Example.** `[↯SN-de/9.2] import group 3 appears after group 4`

**Example.** `[↯SN-de/9.2] import `gossamer` is out of alphabetical order (after `vacuous`)`

**Resolution.** Reorder imports by group and alphabetically.

### `SN-de/9.3` — Group separation

Import groups must be separated by exactly one blank line; no blank
line is permitted within a group.

**Example.** `[↯SN-de/9.3] import groups must be separated by exactly one blank line`

**Example.** `[↯SN-de/9.3] unexpected blank line within an import group`

**Resolution.** Add or remove the blank line as required.

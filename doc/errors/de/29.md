# `SN-de/29` — File naming conventions

Raised by the Decorum compiler plugin when a file's name does not
match a documented Soundness convention.

## Cause

Soundness uses a small set of file-name conventions: types live in
files named after the type, package-level givens live in
`<package>_<scope>.scala` files, and so on. Names outside this set
are rejected.

## Example message

```
[↯SN-de/29] file name `helpers.scala` does not match a documented soundness convention
```

## Resolution

Rename the file to match the convention for its content kind. See
the Decorum documentation for the full list of permitted patterns.

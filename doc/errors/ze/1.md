# `SN-ze/1` — `ParseError`

Raised when Zephyrine cannot parse a value of a registered format.

## Cause

`ParseError` is a generic format-parsing error. It carries the
`Format` being parsed, the `Position` at which the parser was
located, and the `Issue` reported by the format-specific parser.

## Example message

```
[↯SN-ze/1] the JSON was not valid at offset 12 because an unexpected character was found
```

## Resolution

Use the embedded `Position` and `Issue` to locate and correct the
problem in the input.

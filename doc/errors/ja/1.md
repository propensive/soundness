# `SN-ja/1` — `JsonError`

Raised when accessing a JSON value fails because the value is missing
or has the wrong type.

## Cause

Jacinta's `Json` API allows direct field, index, and type access on a
parsed value. When the requested access cannot be satisfied, this
error is thrown. The message is `could not access the value because
$reason`.

## Variants

### `SN-ja/1.1` — `OutOfRange`

An index access referred to a position past the end of a JSON array.

**Example.** `[↯SN-ja/1.1] could not access the value because the array index is out of range`

**Resolution.** Check the array length before indexing, or use an
optional accessor.

### `SN-ja/1.2` — `NotType`

A typed accessor was used but the value's actual primitive type is
different.

**Example.** `[↯SN-ja/1.2] could not access the value because the JSON value had type string instead of number`

**Resolution.** Use the accessor matching the JSON value's actual
type, or coerce/convert the value before reading it.

### `SN-ja/1.3` — `Absent`

The named field does not exist in the JSON object.

**Example.** `[↯SN-ja/1.3] could not access the value because the JSON value was not present`

**Resolution.** Confirm the field name and case; use an optional
accessor if the field may legitimately be missing.

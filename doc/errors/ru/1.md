# `SN-ru/1` — `probe` macro reports a type

Raised at compile time by Rudiments' `probe` macro, which
deliberately halts compilation to report the dealiased, widened
type of an expression.

## Cause

`probe[target]` exists for debugging type inference; it never
succeeds — its purpose is to print the type at compile time.

## Example message

```
[↯SN-ru/1] the type is List[Int]
```

## Resolution

Remove the `probe` call once you have read the type; do not ship
code that contains it.

# `SN-av/3` — `TimezoneError`

Raised when a value does not name a known timezone.

## Cause

Aviation looks timezones up by IANA name (e.g. `Europe/Paris`). When
the name is not present in the loaded TZDB, this error is thrown.

## Example message

```
[↯SN-av/3] the name Mars/Olympus does not refer to a known timezone
```

## Resolution

Use a name from the TZDB; for legacy abbreviations (`PST`, `EST`)
prefer the corresponding IANA zone (`America/Los_Angeles`,
`America/New_York`).

# `SN-av/1` — `TimeError`

Raised when Aviation cannot interpret a date or time value.

## Cause

Aviation's date and calendar APIs flag this error when an input does
not conform to the expected format, refers to a non-existent date in
its calendar, or names a kind that Aviation does not recognise. The
message is `the date was not valid because $reason`.

## Variants

### `SN-av/1.1` — `Format`

The input did not match the date format being parsed.

**Example.** `[↯SN-av/1.1] the date was not valid because 2026-13-01 does not conform to format ISO-8601 (... at ...)`

**Resolution.** Pass an input that matches the format, or choose a
format that matches the input.

### `SN-av/1.2` — `Invalid`

The numeric components form a date that does not exist in the
chosen calendar (e.g. February 30 in the Gregorian calendar).

**Example.** `[↯SN-av/1.2] the date was not valid because 2026-2-30 does not exist in the calendar Gregorian`

**Resolution.** Adjust the day, month, or year to a valid combination.

### `SN-av/1.3` — `Unknown`

A textual value did not name a recognised date kind (e.g. day name,
month name).

**Example.** `[↯SN-av/1.3] the date was not valid because foo is not a recognized weekday`

**Resolution.** Use one of the documented names for the kind.

# `SN-av/2` — `TimestampError`

Raised when Aviation cannot parse a timestamp string.

## Cause

Aviation's timestamp parser expects an ISO-8601-style timestamp.
The constructor records the offending `value` and a `reason: Reason`
indicating which step of the parse failed; the rendered message is
`the time $value could not be parsed because $reason`.

## Variants

### `SN-av/2.1` — `BadFormat`

The input did not match the expected ISO-8601 timestamp pattern at
all.

**Example.** `[↯SN-av/2.1] the time not-a-timestamp could not be parsed because the input did not match the expected timestamp format`

**Resolution.** Provide the timestamp in ISO-8601 form
(`YYYY-MM-DD[T ]HH:MM:SS`); pre-process non-standard inputs.

### `SN-av/2.2` — `BadNumber`

The input matched the timestamp pattern but one of its numeric
components could not be parsed as an integer (a `NumberError` was
raised internally).

**Example.** `[↯SN-av/2.2] the time 2024-XX-15T14:30:59 could not be parsed because a numeric component could not be parsed as an integer`

**Resolution.** Confirm each component (year, month, day, hour,
minute, second) is a plain decimal integer.

### `SN-av/2.3` — `BadTime`

A date or time component was outside its valid range — e.g. month
13, day 32, or hour 25 (a `TimeError` was raised internally).

**Example.** `[↯SN-av/2.3] the time 2024-13-15T14:30:59 could not be parsed because a date or time component was outside the valid range`

**Resolution.** Validate the component values before formatting,
or compute them from a calendar that constrains them to legal
ranges.

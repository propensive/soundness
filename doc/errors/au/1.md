# `SN-au/1` — `PojoError`

Raised when Austronesian cannot deserialize a value from Java POJO
types.

## Cause

Austronesian provides interop with classic Java bean / POJO
hierarchies. When a value cannot be decomposed into the expected
properties — typically because a getter is missing, a type does not
match, or reflection fails — this error is thrown.

## Example message

```
[↯SN-au/1] could not deserialize from Java POJO types
```

## Resolution

Confirm the source class follows JavaBean conventions; ensure the
getters required by the target type exist and are accessible. FIXME:
more specific reasons may be added as the module evolves.

# `SN-159` — `EmailAddressError`

Raised when a value cannot be parsed as a valid email address.

## Cause

Urticose validates email addresses against the syntax rules in
RFC 5321 / 5322 (with practical relaxations). The message is `the
email address is not valid because $reason`.

## Variants

### `SN-159.1` — `Empty`

The value is empty.

**Example.** `[↯SN-159.1] the email address is not valid because it is empty`

**Resolution.** Provide a non-empty address.

### `SN-159.2` — `InvalidDomain`

The domain part is neither a valid hostname nor a valid IP literal.

**Example.** `[↯SN-159.2] the email address is not valid because the domain is not a valid hostname: the character _ is not allowed in a hostname`

**Resolution.** Correct the domain syntax; consult the inner
`HostnameError`/`IpAddressError` variant for specifics.

### `SN-159.3` — `LongLocalPart`

The local part exceeds 64 characters.

**Example.** `[↯SN-159.3] the email address is not valid because the local part is more than 64 characters long`

**Resolution.** Shorten the local part to at most 64 characters.

### `SN-159.4` — `TerminalPeriod`

The local part ends in a period.

**Example.** `[↯SN-159.4] the email address is not valid because the local part ends in a period, which is not allowed`

**Resolution.** Remove the trailing period.

### `SN-159.5` — `SuccessivePeriods`

The local part contains two adjacent periods.

**Example.** `[↯SN-159.5] the email address is not valid because the local part contains two adjacent periods`

**Resolution.** Remove one of the periods, or quote the local part
if successive periods are required.

### `SN-159.6` — `InitialPeriod`

The local part starts with a period.

**Example.** `[↯SN-159.6] the email address is not valid because the local part starts with a period, which is not allowed`

**Resolution.** Remove the leading period.

### `SN-159.7` — `UnescapedQuote`

A double-quote in the local part was not escaped.

**Example.** `[↯SN-159.7] the email address is not valid because the local part contains a quote character which is not escaped`

**Resolution.** Either escape the quote with a backslash or use a
properly quoted local part.

### `SN-159.8` — `UnclosedQuote`

The quoted local part has no closing quote.

**Example.** `[↯SN-159.8] the email address is not valid because the quoted local part has no closing quote`

**Resolution.** Add the missing `"`.

### `SN-159.9` — `MissingDomain`

The address has no domain part.

**Example.** `[↯SN-159.9] the email address is not valid because the domain is missing`

**Resolution.** Append `@` followed by the domain.

### `SN-159.10` — `MissingAtSymbol`

The address contains no `@`.

**Example.** `[↯SN-159.10] the email address is not valid because the at-symbol is missing`

**Resolution.** Insert `@` between the local part and the domain.

### `SN-159.11` — `UnclosedIpAddress`

The domain begins with `[` but does not end with `]`.

**Example.** `[↯SN-159.11] the email address is not valid because the domain begins with [ but does not end with ]`

**Resolution.** Close the IP-literal with `]`.

### `SN-159.12` — `InvalidChar`

The local part contains a character that is not allowed.

**Example.** `[↯SN-159.12] the email address is not valid because the local part contains the character > which is not allowed`

**Resolution.** Remove the offending character or quote the local
part to permit it.

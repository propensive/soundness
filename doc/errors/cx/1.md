# `SN-cx/1` — `BindError`

Raised when Coaxial cannot bind a socket.

## Cause

The OS refused to allocate the requested socket. The constructor
takes a `reason: Reason`; the rendered message is `the socket
could not be bound because $reason`.

## Variants

### `SN-cx/1.1` — `PortInUse`

Another process is already bound to the requested port.

**Example.** `[↯SN-cx/1.1] the socket could not be bound because another process is already bound to the port`

**Resolution.** Free the port (stop the conflicting process) or
choose a different port.

### `SN-cx/1.2` — `PermissionDenied`

The current user does not have permission to bind the requested
port — typically the case for ports below 1024 on POSIX systems.

**Example.** `[↯SN-cx/1.2] the socket could not be bound because the user does not have permission to bind the port`

**Resolution.** Run with elevated privileges, or choose an
unprivileged port.

### `SN-cx/1.3` — `AddressUnavailable`

The requested address is not available on this host (e.g. binding
to an IP that no interface owns).

**Example.** `[↯SN-cx/1.3] the socket could not be bound because the requested address is not available on this host`

**Resolution.** Verify the address is configured on a local
interface, or bind to `0.0.0.0` / `::` for any interface.

# `SN-zp/1` — `ZipError`

Raised when a ZIP-archive operation cannot complete.

## Cause

Zeppelin manages ZIP entries by path. This error reports duplicate
or missing paths and invalid entry names. The message is `the ZIP
operation failed because $reason`.

## Variants

### `SN-zp/1.1` — `DuplicateEntry`

An entry being added has the same path as one already in the
archive.

**Example.** `[↯SN-zp/1.1] the ZIP operation failed because the path /foo/bar.txt is a duplicate entry`

**Resolution.** Choose a different path, or remove the existing
entry first.

### `SN-zp/1.2` — `NotFound`

An operation referenced an entry path that does not exist in the
archive.

**Example.** `[↯SN-zp/1.2] the ZIP operation failed because path /foo/bar.txt was not found in the ZIP file`

**Resolution.** List entries before reading; check for typos in
the path.

### `SN-zp/1.3` — `InvalidName`

An entry name is not valid for a ZIP entry — for example contains
characters disallowed by the format.

**Example.** `[↯SN-zp/1.3] the ZIP operation failed because the name foo\bar is not valid for a ZIP entry`

**Resolution.** Sanitise the name to a valid entry path.

# `SN-no/4` — Bad type in `parse`

Raised at compile time by Nomenclature's `parse` macro when the
type constraint is not in the expected form.

## Cause

The macro expects a specific type-level shape for the constraint
parameter. Other shapes cannot be analysed.

## Example message

```
[↯SN-no/4] Bad type
```

## Resolution

FIXME: the message is intentionally generic. Inspect the type the
macro is being applied to and confirm it matches the documented
form.

# `SN-no/5` — Name fails rule validation at compile time

Raised at compile time by Nomenclature's `parse` macro when a
literal value does not satisfy the rule for the system being
parsed against.

## Cause

The macro evaluates the rule against the literal text. When the
rule rejects it, the macro halts with a message generated from
`rule.describe(text)`.

## Example message

```
[↯SN-no/5] the name is not valid because it contains the prohibited character /
```

## Resolution

Edit the literal so it satisfies the rule.

# `SN-no/1` — `NameError`

Raised when a value does not satisfy the rules of a named-type's
`Nominative` system.

## Cause

Nomenclature defines `Name`-like types parameterised by a
`Nominative` system whose rules constrain valid values. When a
candidate violates a rule, this error is thrown carrying the
`name`, the failing `Rule`, and a `parameter` describing the rule's
specific complaint.

## Example message

```
[↯SN-no/1] the name foo/bar is not valid because it contains the prohibited character /
```

## Resolution

Adjust the value to satisfy the rule, or pick a different
`Nominative` system whose rules permit the value.

# `SN-no/3` — Companion object has wrong type

Raised at compile time by Nomenclature's macro when the companion
object referenced does not have the type the macro expects.

## Cause

The macro inspects the companion module to read its rule set. A
companion of an unexpected type cannot be processed.

## Example message

```
[↯SN-no/3] The companion object did not have the expected type.
```

## Resolution

FIXME: this typically indicates a project-internal misconfiguration;
verify the system type's companion follows Nomenclature's
conventions.

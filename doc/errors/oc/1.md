# `SN-oc/1` ‚Äî `GitError`

Raised when a Git operation invoked through Octogenarian fails. Each
variant identifies the specific Git sub-operation that did not complete.

## Cause

Octogenarian shells out to the `git` executable and inspects its exit
code and output. Any non-success outcome is mapped to one of the
variants below. The error message is constructed from a fixed prefix
plus the `Reason` variant: `the Git operation could not be completed
because $reason`.

## Variants

### `SN-oc/1.1` ‚Äî `CannotExecuteGit`

The `git` command itself could not be executed ‚Äî typically because no
`git` binary was found on the `PATH`, or it is not executable.

**Example.** `[‚ÜØSN-oc/1.1] the Git operation could not be completed because the `git` command could not be executed`

**Resolution.** Install Git, or ensure the `git` binary is on the
`PATH` of the process Octogenarian is running in.

### `SN-oc/1.2` ‚Äî `CloneFailed`

A `git clone` invocation returned a non-zero status.

**Example.** `[‚ÜØSN-oc/1.2] the Git operation could not be completed because the repository could not be cloned`

**Resolution.** Verify the source URL, network access, and any
required credentials. Inspect Git's stderr for the underlying cause.

### `SN-oc/1.3` ‚Äî `InvalidRepoPath`

The path supplied as the working repository is syntactically invalid.

**Example.** `[‚ÜØSN-oc/1.3] the Git operation could not be completed because the repository path was not valid`

**Resolution.** Pass a well-formed path; check for embedded null
bytes or platform-specific path issues.

### `SN-oc/1.4` ‚Äî `RepoDoesNotExist`

The path is well-formed but no Git repository exists there.

**Example.** `[‚ÜØSN-oc/1.4] the Git operation could not be completed because the repository does not exist`

**Resolution.** Initialise the repository (`git init`) or supply the
path of an existing one.

### `SN-oc/1.5` ‚Äî `BranchDoesNotExist`

A branch named in the operation is not present in the repository.

**Example.** `[‚ÜØSN-oc/1.5] the Git operation could not be completed because the branch does not exist`

**Resolution.** Check the branch name (case-sensitive) and ensure it
has been fetched if it lives on a remote.

### `SN-oc/1.6` ‚Äî `CommitDoesNotExist`

A commit referenced by SHA or symbolic name is unknown to the
repository.

**Example.** `[‚ÜØSN-oc/1.6] the Git operation could not be completed because the commit does not exist`

**Resolution.** Verify the commit SHA; fetch from the relevant remote
if the commit lives there.

### `SN-oc/1.7` ‚Äî `CommitFailed`

`git commit` returned non-zero ‚Äî for example because there were no
staged changes, or a hook rejected the commit.

**Example.** `[‚ÜØSN-oc/1.7] the Git operation could not be completed because the commit could not be created`

**Resolution.** Inspect Git's stderr for the underlying cause; ensure
there is something to commit and that any pre-commit hook succeeds.

### `SN-oc/1.8` ‚Äî `CannotSwitchBranch`

A `git checkout`/`git switch` operation failed, typically due to
uncommitted changes in the working tree or an invalid target.

**Example.** `[‚ÜØSN-oc/1.8] the Git operation could not be completed because the branch could not be changed`

**Resolution.** Commit, stash, or discard pending changes; verify the
target branch exists.

### `SN-oc/1.9` ‚Äî `PullFailed`

`git pull` did not complete successfully ‚Äî most often because of a
merge conflict or network failure.

**Example.** `[‚ÜØSN-oc/1.9] the Git operation could not be completed because the pull operation did not complete`

**Resolution.** Resolve any reported conflicts; check connectivity to
the remote.

### `SN-oc/1.10` ‚Äî `BranchFailed`

A new branch could not be created (`git branch` returned non-zero) ‚Äî
typically because the name already exists or is invalid.

**Example.** `[‚ÜØSN-oc/1.10] the Git operation could not be completed because the new branch could not be created`

**Resolution.** Choose a name that is not already in use and conforms
to Git's reference-name rules.

### `SN-oc/1.11` ‚Äî `TagFailed`

A new tag could not be created. The most common reason is that a tag
of that name already exists.

**Example.** `[‚ÜØSN-oc/1.11] the Git operation could not be completed because the new tag could not be created`

**Resolution.** Use a fresh tag name, or delete the existing tag
first.

### `SN-oc/1.12` ‚Äî `AddFailed`

`git add` failed ‚Äî for example the path does not exist, or is
ignored.

**Example.** `[‚ÜØSN-oc/1.12] the Git operation could not be completed because the path could not be added`

**Resolution.** Confirm the file exists relative to the work tree and
is not excluded by `.gitignore`.

### `SN-oc/1.13` ‚Äî `NoWorkTree`

An operation that requires a work tree was attempted on a bare
repository.

**Example.** `[‚ÜØSN-oc/1.13] the Git operation could not be completed because this bare repository does not have a work tree`

**Resolution.** Run the operation against a non-bare clone, or use a
form of the operation that does not need a work tree.

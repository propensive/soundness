# `SN-oc/2` — `GitRefError`

Raised when a value is not a valid Git reference name.

## Cause

Octogenarian validates reference names against `git
check-ref-format`'s rules. The constructor records the offending
`value` and a `reason: Reason`; the rendered message is `$value
is not a valid Git reference because $reason`.

## Variants

### `SN-oc/2.1` — `LeadingOrTrailingDot`

A path segment of the reference starts or ends with `.`.

**Example.** `[↯SN-oc/2.1] .foo is not a valid Git reference because a path segment starts or ends with .`

**Resolution.** Trim the leading or trailing dot from the segment.

### `SN-oc/2.2` — `ReservedSuffix`

A path segment ends with `.lock`, which Git reserves for lock
files.

**Example.** `[↯SN-oc/2.2] feature.lock is not a valid Git reference because a path segment ends with the reserved suffix .lock`

**Resolution.** Choose a name without the `.lock` suffix.

### `SN-oc/2.3` — `ReservedSequence`

The name contains `@{`, which Git reserves for reflog selectors
(e.g. `HEAD@{1}`).

**Example.** `[↯SN-oc/2.3] foo@{bar} is not a valid Git reference because the name contains the reserved sequence @{`

**Resolution.** Remove or replace the `@{` sequence.

### `SN-oc/2.4` — `DoubleDot`

The name contains `..`, which Git reserves for range selectors.

**Example.** `[↯SN-oc/2.4] foo..bar is not a valid Git reference because the name contains ..`

**Resolution.** Replace the `..` with a single `.` or another
separator.

### `SN-oc/2.5` — `EmptySegment`

The name contains an empty path segment — typically a leading,
trailing, or doubled `/`.

**Example.** `[↯SN-oc/2.5] foo//bar is not a valid Git reference because the name contains an empty path segment`

**Resolution.** Remove the empty segment.

### `SN-oc/2.6` — `InvalidCharacter`

The name contains a character Git forbids in references: any of
`*`, `[`, `\`, ` `, `^`, `~`, `:`, or `?`.

**Example.** `[↯SN-oc/2.6] foo bar is not a valid Git reference because the name contains a character forbidden by Git`

**Resolution.** Replace the forbidden character with `-` or `_`.

### `SN-oc/2.7` — `BadHash`

The value passed to `GitHash` was not a 40-character lowercase
hexadecimal string.

**Example.** `[↯SN-oc/2.7] abc is not a valid Git reference because the value is not a 40-character hexadecimal hash`

**Resolution.** Provide the full 40-character SHA-1 commit hash.

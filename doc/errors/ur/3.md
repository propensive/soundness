# `SN-ur/3` — `IpAddressError`

Raised when a value cannot be parsed as a valid IPv4 or IPv6 address.

## Cause

Urticose validates both IPv4 (four 0–255 bytes) and IPv6 (eight
hex groups, optionally compressed with `::`) addresses. The message
is `the IP address is not valid because $reason`.

## Variants

### `SN-ur/3.1` — `Ipv4ByteOutOfRange`

An IPv4 byte was numeric but outside the range 0–255.

**Example.** `[↯SN-ur/3.1] the IP address is not valid because the number 300 is not in the range 0-255`

**Resolution.** Use only values 0–255 for each IPv4 byte.

### `SN-ur/3.2` — `Ipv4ByteNotNumeric`

An IPv4 component was not a number.

**Example.** `[↯SN-ur/3.2] the IP address is not valid because the part abc is not a number`

**Resolution.** Use four decimal numbers separated by `.`.

### `SN-ur/3.3` — `Ipv4WrongNumberOfGroups`

The address did not have exactly four `.`-separated groups.

**Example.** `[↯SN-ur/3.3] the IP address is not valid because the address contains 3 period-separated groups instead of 4`

**Resolution.** Provide exactly four numeric groups.

### `SN-ur/3.4` — `Ipv6GroupWrongLength`

An IPv6 group is more than four hex characters long.

**Example.** `[↯SN-ur/3.4] the IP address is not valid because the group is more than 4 hexadecimal characters long`

**Resolution.** Each IPv6 group must be 1–4 hex characters; trim
leading zeroes if necessary.

### `SN-ur/3.5` — `Ipv6GroupNotHex`

An IPv6 group contains a non-hexadecimal character.

**Example.** `[↯SN-ur/3.5] the IP address is not valid because the group 'xyz1' is not a hexadecimal number`

**Resolution.** Use only `0`-`9` and `a`-`f`/`A`-`F` in IPv6 groups.

### `SN-ur/3.6` — `Ipv6TooManyNonzeroGroups`

The address has more non-zero groups than 8 — indicating a malformed
input.

**Example.** `[↯SN-ur/3.6] the IP address is not valid because the address has 9 non-zero groups, which is more than is permitted`

**Resolution.** IPv6 has at most 8 groups; check the input.

### `SN-ur/3.7` — `Ipv6WrongNumberOfGroups`

After expansion of `::`, the address does not have exactly 8 groups.

**Example.** `[↯SN-ur/3.7] the IP address is not valid because the address has 7 groups, but should have 8`

**Resolution.** Add the missing group, or use `::` to compress
consecutive zero groups.

### `SN-ur/3.8` — `Ipv6MultipleDoubleColons`

The address contains `::` more than once, which would be ambiguous.

**Example.** `[↯SN-ur/3.8] the IP address is not valid because :: appears more than once`

**Resolution.** Use `::` at most once per address; expand any extras
to explicit `0` groups.

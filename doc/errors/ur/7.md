# `SN-ur/7` — `UrlError`

Raised when a value cannot be parsed as a valid URL.

## Cause

Urticose validates URL syntax against the components it expects: a
scheme, optional authority (with hostname and port), path, query and
fragment. Each variant pinpoints where parsing failed. The message
includes the bad input and the offset of the failure: `the URL $text
is not valid: $reason at $offset`.

## Variants

### `SN-ur/7.1` — `Expected`

A specific token was expected at the offset, but a different one
appeared. The expectation is one of `Colon`, `More`, `LowerCaseLetter`,
`PortRange`, or `Number`.

**Example.** `[↯SN-ur/7.1] the URL https://example is not valid: a colon was expected at 14`

**Resolution.** Adjust the URL to provide the expected token at the
indicated offset.

### `SN-ur/7.2` — `BadHostname`

The hostname component of the URL is not a valid DNS hostname.

**Example.** `[↯SN-ur/7.2] the URL http://exa_mple/ is not valid: exa_mple was not valid because the character _ is not allowed in a hostname at 7`

**Resolution.** Replace the hostname with a syntactically valid one;
see `SN-ur/2` for hostname rules.

### `SN-ur/7.3` — `BadIpv6`

The bracketed IPv6 literal in the host component is not valid.

**Example.** `[↯SN-ur/7.3] the URL http://[::1::]/ is not valid: the IPv6 address is not valid because :: appears more than once at 7`

**Resolution.** Provide a valid IPv6 literal; see `SN-ur/3` for
IPv6 rules.

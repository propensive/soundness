# `SN-ur/4` — `MacAddressError`

Raised when a value cannot be parsed as a valid MAC address.

## Cause

Urticose expects MAC addresses as six colon-separated hexadecimal
groups of two digits, e.g. `01:23:45:67:89:ab`. The message is `the
MAC address is not valid because $reason`.

## Variants

### `SN-ur/4.1` — `WrongGroupCount`

The address does not have six colon-separated groups.

**Example.** `[↯SN-ur/4.1] the MAC address is not valid because there should be six colon-separated groups, but there were 5`

**Resolution.** Use exactly six groups joined by `:`.

### `SN-ur/4.2` — `WrongGroupLength`

A group is not exactly two hex digits long.

**Example.** `[↯SN-ur/4.2] the MAC address is not valid because group 3 should be two hex digits, but its length is 3`

**Resolution.** Pad single-digit groups with a leading zero, or trim
extra digits.

### `SN-ur/4.3` — `NotHex`

A group contains characters that are not hex digits.

**Example.** `[↯SN-ur/4.3] the MAC address is not valid because group 4 should be a two-digit hex number, but it is xy`

**Resolution.** Use only `0`-`9` and `a`-`f`/`A`-`F`.

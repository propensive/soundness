# `SN-ob/2` — `FrameError`

Raised when Obligatory cannot deframe an incoming message.

## Cause

Obligatory's framing parsers (`ContentLength`, `LengthPrefix`,
`CrLf`) read a byte stream into discrete frames. When the bytes do
not match the expected framing, this error is raised. The
constructor takes a `reason: Reason`; the rendered message is
`could not deframe the message because $reason`.

## Variants

### `SN-ob/2.1` — `ShortRead`

The input ended before a complete frame had been read.

**Example.** `[↯SN-ob/2.1] could not deframe the message because the input ended before a complete frame could be read`

**Resolution.** Confirm the producer wrote a complete frame; check
for a closed connection mid-message.

### `SN-ob/2.2` — `MissingLineFeed`

A carriage return (`\r`) was not followed by a line feed (`\n`)
where the framing required CR-LF.

**Example.** `[↯SN-ob/2.2] could not deframe the message because a carriage return was not followed by a line feed`

**Resolution.** Verify the producer is using CR-LF line endings;
LF-only or CR-only inputs will fail this parser.

### `SN-ob/2.3` — `MalformedLength`

The `Content-Length` header value could not be parsed as an
integer.

**Example.** `[↯SN-ob/2.3] could not deframe the message because the content-length header value could not be parsed`

**Resolution.** Inspect the header bytes; the value must be a plain
decimal integer.

### `SN-ob/2.4` — `UnknownHeader`

A header was encountered that the framing parser does not
recognise (only `Content-Type` and `Content-Length` are
permitted).

**Example.** `[↯SN-ob/2.4] could not deframe the message because an unrecognized header was encountered`

**Resolution.** Strip non-framing headers from the message before
feeding it to the parser; or extend the parser to accept the
header.

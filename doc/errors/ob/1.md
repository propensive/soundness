# `SN-ob/1` — `SseError`

Raised when Obligatory cannot parse a server-sent event.

## Cause

SSE messages are framed as `field: value` lines, with `field`
restricted to `event`, `data`, `id`, or `retry`. The `SseSource`
class also enforces a replay-buffer capacity. When any of these
expectations is violated, this error is raised. The constructor
takes a `reason: Reason`; the rendered message is `the
server-sent event was not valid because $reason`.

## Variants

### `SN-ob/1.1` — `MalformedField`

A line in the SSE stream did not contain the required `:`
separator between field name and value.

**Example.** `[↯SN-ob/1.1] the server-sent event was not valid because a line did not contain the expected field: value separator`

**Resolution.** Verify the producer is emitting valid SSE frames;
each non-empty line must contain a colon.

### `SN-ob/1.2` — `UnknownField`

A field name other than `event`, `data`, `id`, or `retry` was
encountered.

**Example.** `[↯SN-ob/1.2] the server-sent event was not valid because the field name was not one of event, data, id, or retry`

**Resolution.** Strip non-standard fields before parsing; or extend
the parser to accept the field.

### `SN-ob/1.3` — `BadRetryValue`

The `retry` field's value was not a parseable integer.

**Example.** `[↯SN-ob/1.3] the server-sent event was not valid because the retry field value could not be parsed as an integer`

**Resolution.** The `retry` value must be a plain decimal integer
(milliseconds).

### `SN-ob/1.4` — `CapacityExceeded`

The replay range requested from `SseSource.stream(start)` was
larger than the source's circular buffer capacity, so older events
have been overwritten.

**Example.** `[↯SN-ob/1.4] the server-sent event was not valid because the requested replay range exceeded the source buffer capacity`

**Resolution.** Increase the source's `capacity`, or request a more
recent `start` index.

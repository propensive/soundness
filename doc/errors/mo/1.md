# `SN-mo/1` — `SerializationError`

Raised when Monotonous cannot deserialize a binary value because of
an invalid byte at a known position.

## Cause

Monotonous reports the offending `char` and its `position` so the
location of the corruption can be identified.

## Example message

```
[↯SN-mo/1] the character ? at 12 was not valid
```

## Resolution

Inspect the input around the indicated position; check the
encoding of the source.

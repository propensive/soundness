# `SN-cs/1` — `DsvError`

Raised when Caesura cannot parse a row of delimiter-separated values.

## Cause

Caesura parses CSV/TSV/DSV against a `DsvFormat`. The parser raises
this error on malformed input. The message is `could not parse row
data because $reason`.

## Variants

### `SN-cs/1.1` — `MisplacedQuote`

A quote character appeared after the start of a cell, where the
parser was not expecting one.

**Example.** `[↯SN-cs/1.1] could not parse row data because a quote was found after the start of a cell`

**Resolution.** Either escape the quote according to the format's
quoting rules, or quote the entire field.

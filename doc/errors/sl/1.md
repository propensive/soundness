# `SN-sl/1` — `RemoteError`

Raised when Superlunary fails to perform a remote operation.

## Cause

Superlunary marshals a call to a remote process and reads the result
back. Any failure in serialising the call, deserialising the result,
or otherwise transporting it produces this error. The message is
`failed to perform a remote operation because $reason`.

## Variants

### `SN-sl/1.1` — `Serialization`

The output (the call or its arguments) could not be serialised for
transmission.

**Example.** `[↯SN-sl/1.1] failed to perform a remote operation because the output could not be serialized`

**Resolution.** Confirm that all argument types have the necessary
encoders/decoders in scope and produce well-formed bytes.

### `SN-sl/1.2` — `Deserialization`

The response from the remote could not be deserialised.

**Example.** `[↯SN-sl/1.2] failed to perform a remote operation because the input could not be deserialized`

**Resolution.** Verify the remote is producing the expected payload
shape; mismatched schemas between client and server are the typical
cause.

### `SN-sl/1.3` — `Unknown`

The cause does not match either of the more specific variants.

**Example.** `[↯SN-sl/1.3] failed to perform a remote operation because of an unknown reason`

**Resolution.** Inspect logs and any cause chain.

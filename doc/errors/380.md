# `SN-380` — Type has multiple `map` methods

Raised at compile time by Mercator's `functor` macro when the
target type has more than one `map` overload, and the macro cannot
choose between them.

## Cause

Multiple `map` overloads are ambiguous for derivation purposes.

## Example message

```
[↯SN-380] the type Foo has more than one possible map method
```

## Resolution

Remove or rename overloads, or supply a `Functor` instance manually.

# `SN-ca/1` — `FeedError`

Raised when Cacophony cannot record from an audio feed.

## Cause

Cacophony opens an audio capture line and reads samples from it. This
error reports a problem with the line itself, the requested
configuration, or operations performed after recording has stopped.
The message is `could not record from feed $feed because $reason`.

## Variants

### `SN-ca/1.1` — `Unavailable`

The requested audio line could not be opened — typically because no
input device is connected, or another process has exclusive use of
it.

**Example.** `[↯SN-ca/1.1] could not record from feed default because the audio line could not be opened`

**Resolution.** Ensure an input device is connected and that the
process has permission to use it.

### `SN-ca/1.2` — `UnsupportedConfiguration`

The combination of sample rate, channel count, and sample size was
not supported by the device.

**Example.** `[↯SN-ca/1.2] could not record from feed default because the requested configuration is not supported`

**Resolution.** Choose a configuration the device supports; query
its capabilities and pick a compatible format.

### `SN-ca/1.3` — `Closed`

An operation was attempted after recording was stopped.

**Example.** `[↯SN-ca/1.3] could not record from feed default because the recording has already been stopped`

**Resolution.** Open a fresh recording rather than re-using a
stopped one.

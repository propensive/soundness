# `SN-pn/1` — `FontError`

Raised when Phoenicia cannot read a font file.

## Cause

Phoenicia parses font files (typically TrueType/OpenType) and reports
this error when the file is missing required tables, uses an unknown
format, or fails the magic-number check. The message is `the font
could not be read because $reason`.

## Variants

### `SN-pn/1.1` — `MissingTable`

A required table identified by `tag` was not found in the font file.

**Example.** `[↯SN-pn/1.1] the font could not be read because the table cmap was not found`

**Resolution.** Use a font that contains the required tables; many
font subsetting tools can strip mandatory tables.

### `SN-pn/1.2` — `UnknownFormat`

A table was located but its content does not match any version of
the format Phoenicia knows how to read.

**Example.** `[↯SN-pn/1.2] the font could not be read because the table contains data in an unknown format`

**Resolution.** Use a font that conforms to a supported version of
the OpenType/TrueType specification.

### `SN-pn/1.3` — `MagicNumber`

The font's leading bytes did not match the expected magic number,
indicating the file is not a font Phoenicia recognises.

**Example.** `[↯SN-pn/1.3] the font could not be read because the font did not contain expected check data`

**Resolution.** Confirm the file is a TrueType or OpenType font and
not corrupted; check that the file was opened in binary mode.

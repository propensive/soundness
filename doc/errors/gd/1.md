# `SN-gd/1` ‚Äî `GeolocationError`

Raised when Geodesy cannot parse a value as a `geo:` URI.

## Cause

Geodesy parses RFC 5870 geo URIs of the form
`geo:<lat>,<lon>[,<alt>][;<param>=<value>]‚Ä¶`. If any part of the URI
fails to satisfy the syntax, this error is thrown with the variant
identifying which part. The message is `the geo URI is not in the
correct format because $reason`.

## Variants

### `SN-gd/1.1` ‚Äî `MissingEquals`

A parameter component had no `=` to separate name from value.

**Example.** `[‚ÜØSN-gd/1.1] the geo URI is not in the correct format because the parameter does not contain an `=``

**Resolution.** Write parameters as `name=value`.

### `SN-gd/1.2` ‚Äî `MultipleEquals`

A single parameter contained more than one `=`.

**Example.** `[‚ÜØSN-gd/1.2] the geo URI is not in the correct format because the parameter contains more than one `=``

**Resolution.** Percent-encode `=` characters that should appear in
the value.

### `SN-gd/1.3` ‚Äî `BadScheme`

The URI did not start with `geo:`.

**Example.** `[‚ÜØSN-gd/1.3] the geo URI is not in the correct format because the value does not begin with the `geo:` URI scheme`

**Resolution.** Prefix the URI with `geo:`.

### `SN-gd/1.4` ‚Äî `ExpectedSemicolon`

After the altitude value, a `;` was expected but not found.

**Example.** `[‚ÜØSN-gd/1.4] the geo URI is not in the correct format because a `;` was expected after the altitude value`

**Resolution.** Separate altitude from any subsequent parameters with
`;`.

### `SN-gd/1.5` ‚Äî `UnexpectedSuffix`

Content followed the coordinates that began with neither `,` nor `;`.

**Example.** `[‚ÜØSN-gd/1.5] the geo URI is not in the correct format because a `,` or `;` was expected`

**Resolution.** Use `,` to introduce altitude and `;` to introduce
parameters.

### `SN-gd/1.6` ‚Äî `ExpectedCoordinates`

Latitude and longitude were missing or could not be parsed as
numbers.

**Example.** `[‚ÜØSN-gd/1.6] the geo URI is not in the correct format because latitude and longitude coordinates were expected`

**Resolution.** Provide two comma-separated decimal numbers
immediately after `geo:`.

### `SN-gd/1.7` ‚Äî `BadUncertainty`

The value of a `u=` (uncertainty) parameter was not a valid number.

**Example.** `[‚ÜØSN-gd/1.7] the geo URI is not in the correct format because the `uncertainty` parameter vas not a valid number`

**Resolution.** Provide a non-negative decimal value for `u=`.

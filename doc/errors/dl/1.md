# `SN-dl/1` — `NumberError`

Raised when Distillate cannot decode a text value into a numeric
type.

## Cause

Distillate's `Decodable in Text` instances for numeric primitives
parse the input via the JDK's number parsers, then range-check
against the target type's bounds. The constructor records the
offending `text`, the `specializable` (target type), and a
`reason: Reason`; the rendered message is `$text is not a valid
$specializable because $reason`.

## Variants

### `SN-dl/1.1` — `Unparseable`

The input was not a syntactically valid number for the target
type — `Integer.parseInt` (or sibling) threw a
`NumberFormatException`.

**Example.** `[↯SN-dl/1.1] abc is not a valid Int because the input could not be parsed as a number`

**Resolution.** Sanitise the input before decoding, or guard
against non-numeric values.

### `SN-dl/1.2` — `OutOfRange`

The input was a syntactically valid number but fell outside the
target type's representable range — currently raised by the
`Byte` and `Short` decoders when the integer is outside their
bounds.

**Example.** `[↯SN-dl/1.2]300 is not a valid Byte because the input was outside the target type's representable range`

**Resolution.** Decode into a wider type (e.g. `Int`), or
clamp the value before decoding.

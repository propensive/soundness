# `SN-hp/2` — `DivisionError`

Raised when a division produces a result that cannot be represented
in the target type.

## Cause

Hypotenuse's checked division raises this error for cases such as
integer division by zero or `MinValue / -1` (which overflows the
positive range).

## Example message

```
[↯SN-hp/2] the result is unrepresentable
```

## Resolution

Guard against zero divisors; for the `MinValue / -1` case, widen
the dividend before dividing.

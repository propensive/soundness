# `SN-hp/1` — `OverflowError`

Raised when a numeric operation would exceed the representable
range of its result type.

## Cause

Hypotenuse's checked arithmetic raises this error rather than
silently wrapping when an addition, subtraction, multiplication,
or shift would overflow.

## Example message

```
[↯SN-hp/1] an overflow error occurred
```

## Resolution

Use a wider numeric type for the result, or restructure the
computation to avoid intermediate overflow.

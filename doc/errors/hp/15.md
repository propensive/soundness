# `SN-hp/15` — Bound expression must use a comparison operator

Raised at compile time by Hypotenuse's `bounded` macro when the
bound expression does not use one of the recognised comparison
operators.

## Cause

The macro analyses bound expressions of the shape
`x <= n`/`x < n`/`x >= n`/`x > n`. Anything else cannot be turned
into a refinement.

## Example message

```
[↯SN-hp/15] <message describing the unsupported bound shape>
```

## Resolution

Express the bound using `<`, `<=`, `>`, or `>=` against a constant.

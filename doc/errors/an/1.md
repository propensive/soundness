# `SN-an/1` — `DataError`

Raised when an Anamnesis database operation cannot proceed.

## Cause

Anamnesis maintains an in-memory store of references and the
relations between them. When a query refers to data the store
does not know about, this error is raised. The constructor takes
a `reason: Reason`; the rendered message is `the database
operation failed because $reason`.

## Variants

### `SN-an/1.1` — `UnknownReference`

The value passed to `Database.ref` has not previously been
stored, so no `Ref` can be returned for it.

**Example.** `[↯SN-an/1.1] the database operation failed because the value has not been stored in the database`

**Resolution.** Call `Database.store` with the value before
asking for a `Ref` to it.

# `SN-am/4` — No working directory

Raised when Ambience cannot determine the current working directory.

## Cause

Ambience reads the working directory from the JVM's `user.dir`
property. When that lookup fails or returns nothing, this error is
thrown.

## Example message

```
[↯SN-am/4] there is no working directory
```

## Resolution

Run the JVM in a context where `user.dir` is set, or use an
`Optional` accessor.

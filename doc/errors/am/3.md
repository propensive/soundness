# `SN-am/3` — System property not defined

Raised when Ambience is asked for a Java system property that is not
set.

## Cause

Ambience reads system properties via the `Properties` typeclass.
When `System.getProperty` returns no value for the requested name,
this error is thrown.

## Example message

```
[↯SN-am/3] the system property java.home was not defined
```

## Resolution

Either define the property (with `-D…=…` on the JVM command line, or
in code before the read) or use an `Optional` accessor.

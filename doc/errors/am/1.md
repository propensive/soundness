# `SN-am/1` — Environment variable not defined

Raised when an environment variable required by the program is not
defined in the current `Environment`.

## Cause

Ambience looks up environment variables via the `Environment` typeclass.
If the variable named is absent — i.e. `System.getenv` returns no value —
this error is thrown.

## Example message

```
[↯SN-am/1] the environment variable HOME was not defined
```

## Resolution

Set the environment variable before running the program, or provide a
default by using `Optional` access methods rather than the strict ones.

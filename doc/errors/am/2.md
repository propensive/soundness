# `SN-am/2` — No home directory

Raised when Ambience cannot determine a home directory for the
current user.

## Cause

Ambience reads the home directory from the platform's `HOME`/USER
profile facilities. When that lookup fails — typically because the
relevant environment variable is unset — this error is thrown.

## Example message

```
[↯SN-am/2] there is no home directory
```

## Resolution

Set the environment so a home directory can be resolved (`HOME` on
POSIX, `USERPROFILE` on Windows), or use an `Optional` accessor that
tolerates the missing value.

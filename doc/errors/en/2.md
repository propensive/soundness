# `SN-en/2` — `PemError`

Raised when Enigmatic cannot parse PEM-encoded content.

## Cause

PEM content has the form `-----BEGIN <label>-----` / Base64 body /
`-----END <label>-----`. This error flags structural problems before
the body is decoded. The message is `could not parse PEM content
because $reason`.

## Variants

### `SN-en/2.1` — `BeginMissing`

The opening `-----BEGIN <label>-----` line was not found.

**Example.** `[↯SN-en/2.1] could not parse PEM content because the BEGIN line could not be found`

**Resolution.** Confirm the input is PEM-encoded; some sources
strip the BEGIN/END lines and require re-wrapping.

### `SN-en/2.2` — `EndMissing`

The closing `-----END <label>-----` line was not found.

**Example.** `[↯SN-en/2.2] could not parse PEM content because the END line could not be found`

**Resolution.** Confirm the input is complete and not truncated.

### `SN-en/2.3` — `BadBase64`

The body between BEGIN and END could not be decoded as Base64.

**Example.** `[↯SN-en/2.3] could not parse PEM content because could not parse the BASE-64 PEM message`

**Resolution.** Inspect the body for invalid characters or line
endings; some pipelines mangle Base64 content.

### `SN-en/2.4` — `EmptyFile`

The input was empty.

**Example.** `[↯SN-en/2.4] could not parse PEM content because the file was empty`

**Resolution.** Provide a non-empty PEM file.

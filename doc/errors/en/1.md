# `SN-en/1` — `CryptoError`

Raised when Enigmatic cannot decode encrypted data.

## Cause

The cryptographic primitive (cipher, signature, or MAC) rejected
the input — typically due to an invalid key, an authentication-tag
mismatch, or malformed ciphertext. The constructor records a
`detail` message describing the specific failure.

## Example message

```
[↯SN-en/1] could not decode the encrypted data: bad authentication tag
```

## Resolution

Confirm the key and IV match those used at encryption time; verify
the ciphertext has not been truncated or modified in transit.

# `SN-en/1` — `CryptoError`

Raised when Enigmatic cannot decode encrypted data.

## Cause

The cryptographic primitive (cipher, signature, or MAC) rejected the
input. The constructor takes a `reason: Reason` indicating why the
operation failed; the rendered message is `could not decode the
encrypted data because $reason`.

## Variants

### `SN-en/1.1` — `BadPadding`

The padding bytes did not match the cipher's expectations — typically
because the ciphertext was corrupted or the wrong key was supplied
(authentication-tag mismatch in AEAD modes).

**Example.** `[↯SN-en/1.1] could not decode the encrypted data because the ciphertext was corrupted or the key was wrong`

**Resolution.** Confirm the key and IV match those used at encryption
time; verify the ciphertext has not been truncated or modified in
transit.

### `SN-en/1.2` — `IllegalBlockSize`

The input length was not a multiple of the cipher's block size, or
otherwise incompatible with the cipher's framing.

**Example.** `[↯SN-en/1.2] could not decode the encrypted data because the input length was not valid for the cipher`

**Resolution.** Verify the ciphertext is complete and was produced by
the same cipher and mode now being used to decrypt it.

### `SN-en/1.3` — `InvalidKey`

The key bytes were rejected by the cipher — wrong size, invalid
format, or unsupported parameters.

**Example.** `[↯SN-en/1.3] could not decode the encrypted data because the key was not valid for the cipher`

**Resolution.** Confirm the key was produced for the same cipher and
parameter set.

### `SN-en/1.4` — `InvalidAlgorithm`

The requested algorithm or mode is not available in the JVM's
configured providers.

**Example.** `[↯SN-en/1.4] could not decode the encrypted data because the algorithm was not available`

**Resolution.** Install or enable a JCA provider that supports the
algorithm; check `java.security` policy.

### `SN-en/1.5` — `IoFailure`

The cipher implementation could not read or write the data buffers —
typically a low-level I/O fault.

**Example.** `[↯SN-en/1.5] could not decode the encrypted data because data could not be read or written`

**Resolution.** Inspect the cause chain for the underlying I/O error.

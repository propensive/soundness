# `SN-cg/11` — Unrecognised `Unapply`

Raised by Contingency's pattern-type extractor when an `Unapply`
pattern's declared type cannot be processed.

## Cause

Custom extractors expose their result type via the `unapply`
method's signature. When that signature does not match what the
macro expects, this halt fires with the symbol's declared type.

## Example message

```
[↯SN-cg/11] Unapply <type>
```

## Resolution

Use a case-class pattern instead, or adjust the extractor to expose
a recognised result type.

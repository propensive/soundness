# `SN-cg/2` — `ExpectationError`

Raised when an expression that was expected to fail with an `Error`
instead succeeded.

## Cause

Used by Contingency's expectation utilities (e.g. test helpers that
assert a specific error). When the wrapped expression evaluates
without throwing, `ExpectationError` is thrown carrying the result.

## Example message

```
[↯SN-cg/2] the expression was expected to fail, but succeeded
```

## Resolution

Either adjust the test/expectation to match the actual behaviour or
fix the code so that it really does fail.

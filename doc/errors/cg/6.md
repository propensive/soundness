# `SN-cg/6` — Pattern with repetition not exhaustive

Raised by Contingency's pattern-analysis macro when a pattern that
involves repeated parameters does not cover all cases.

## Cause

The macro determines whether a partial-function pattern handles
every possible error type. When the pattern uses repetition (e.g.
varargs) and leaves cases unhandled, this halt fires.

## Example message

```
[↯SN-cg/6] the pattern (which involves repeated parameters) is not exhaustive
```

## Resolution

Add cases to cover the missing alternatives, or restructure the
pattern to avoid repetition.

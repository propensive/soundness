# `SN-cg/12` — Pattern is not a distinct `Error` type

Raised by Contingency's pattern-type extractor when a pattern does
not correspond to a distinct `Error` subtype.

## Cause

Recovery patterns must each pick out a specific `Error` subclass,
because they are used to drive type-level reasoning about which
errors remain after handling. A pattern matching no specific
`Error` type cannot be used.

## Example message

```
[↯SN-cg/12] this pattern could not be recognized as a distinct `Error` type
```

## Resolution

Match against a concrete `Error` subclass, e.g. `error:
HostnameError`.

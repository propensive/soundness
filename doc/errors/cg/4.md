# `SN-cg/4` — `Validation`

Raised when one or more validation messages are collected against
specific document or structure pointers.

## Cause

`Validation` carries a list of `(Pointer, Message)` pairs, each
identifying a location and the issue at that location. It is used
in pipelines that walk a structure and accumulate failures rather
than aborting on the first.

## Example message

```
[↯SN-cg/4] /name: the value was not set; /age: the value was not in range
```

## Resolution

Iterate over the embedded pairs to surface each message at its
pointer; correct the source structure.

# `SN-cg/3` — `Errors`

Raised when several errors must be reported together, keyed by name.

## Cause

`Errors` aggregates multiple `Error` instances into a single
exception, each tagged with a `Text` key. It is used in places
where validation collects every problem before failing — for
example multi-field form validation.

## Example message

The message is composed from the formatted contents of the
underlying error map, e.g.:

```
[↯SN-cg/3] name: the value was not set; email: the email address is not valid because the at-symbol is missing
```

## Resolution

Inspect the underlying map (`Errors.errorMap`) to handle each error
individually.

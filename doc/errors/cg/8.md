# `SN-cg/8` — Unrecognised pattern type

Raised by Contingency's pattern-analysis macro when an extractor
pattern's structure does not match the recogniser.

## Cause

The macro decomposes patterns into the case-class fields it expects
to see. Some extractor shapes (custom `unapply`s, complex
projections) are not recognised.

## Example message

```
[↯SN-cg/8] this type of pattern is not recognized
```

## Resolution

Use a simpler pattern, or destructure manually before invoking
Contingency's pattern-aware operations.

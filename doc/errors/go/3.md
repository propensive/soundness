# `SN-go/3` — Invalid ASCII character literal

Raised at compile time by Gossamer's ASCII literal macro when the
character is not in the ASCII range.

## Cause

Gossamer constrains certain literal forms (e.g. ASCII strings) to
the 7-bit ASCII subset. Non-ASCII characters are rejected.

## Example message

```
[↯SN-go/3] é is not a valid ASCII character
```

## Resolution

Use only ASCII characters in the literal, or use the corresponding
non-ASCII-aware type instead.

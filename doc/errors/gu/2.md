# `SN-gu/2` — `PidError`

Raised when an operation references a process by PID and that
process is not running.

## Cause

Sending a signal or otherwise interacting with a missing PID — for
example because the process has already exited — produces this
error.

## Example message

```
[↯SN-gu/2] the process with PID 4242 is not running
```

## Resolution

Check the process is still alive (e.g. with `ps`) before
interacting; handle the error if races are possible.

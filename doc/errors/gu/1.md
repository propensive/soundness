# `SN-gu/1` — `ExecError`

Raised when a Guillotine command exits with a non-zero status.

## Cause

Guillotine wraps process execution; this error is thrown when the
exit status indicates failure. The constructor captures the
`Command`, the standard-output stream, and the standard-error
stream so callers can inspect both.

## Example message

```
[↯SN-gu/1] execution of the command `git status` failed
```

## Resolution

Inspect `stderr` for the underlying error; check command syntax,
arguments, and whether the binary exists.

# `SN-cr/3` — Upper bound must be a `Double` singleton literal

Raised by Cardinality's `NumericRange` macro when the upper bound
type is not a `Double` singleton literal type.

## Cause

The macro inspects the type of the upper bound and expects a
`ConstantType(DoubleConstant(_))`. Any other shape — for example a
non-singleton type or a non-`Double` literal — triggers this halt.

## Example message

```
[↯SN-cr/3] the upper bound must be a Double singleton literal types
```

## Resolution

Provide a `Double` singleton literal type as the upper bound (e.g.
`5.0` rather than `5`).

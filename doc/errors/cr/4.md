# `SN-cr/4` — Lower bound must be a `Double` singleton literal

Raised by Cardinality's `NumericRange` macro when the lower bound
type is not a `Double` singleton literal type.

## Cause

The macro inspects the type of the lower bound and expects a
`ConstantType(DoubleConstant(_))`. Any other shape triggers this
halt.

## Example message

```
[↯SN-cr/4] the lower bound must be a Double singleton literal types
```

## Resolution

Provide a `Double` singleton literal type as the lower bound (e.g.
`0.0` rather than `0`).

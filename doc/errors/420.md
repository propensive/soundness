# `SN-420` Рђћ `GeolocationError`

Raised when Geodesy cannot parse a value as a `geo:` URI.

## Cause

Geodesy parses RFC 5870 geo URIs of the form
`geo:<lat>,<lon>[,<alt>][;<param>=<value>]Рђд`. If any part of the URI
fails to satisfy the syntax, this error is thrown with the variant
identifying which part. The message is `the geo URI is not in the
correct format because $reason`.

## Variants

### `SN-420.1` Рђћ `MissingEquals`

A parameter component had no `=` to separate name from value.

**Example.** `[Рє»SN-420.1] the geo URI is not in the correct format because the parameter does not contain an `=``

**Resolution.** Write parameters as `name=value`.

### `SN-420.2` Рђћ `MultipleEquals`

A single parameter contained more than one `=`.

**Example.** `[Рє»SN-420.2] the geo URI is not in the correct format because the parameter contains more than one `=``

**Resolution.** Percent-encode `=` characters that should appear in
the value.

### `SN-420.3` Рђћ `BadScheme`

The URI did not start with `geo:`.

**Example.** `[Рє»SN-420.3] the geo URI is not in the correct format because the value does not begin with the `geo:` URI scheme`

**Resolution.** Prefix the URI with `geo:`.

### `SN-420.4` Рђћ `ExpectedSemicolon`

After the altitude value, a `;` was expected but not found.

**Example.** `[Рє»SN-420.4] the geo URI is not in the correct format because a `;` was expected after the altitude value`

**Resolution.** Separate altitude from any subsequent parameters with
`;`.

### `SN-420.5` Рђћ `UnexpectedSuffix`

Content followed the coordinates that began with neither `,` nor `;`.

**Example.** `[Рє»SN-420.5] the geo URI is not in the correct format because a `,` or `;` was expected`

**Resolution.** Use `,` to introduce altitude and `;` to introduce
parameters.

### `SN-420.6` Рђћ `ExpectedCoordinates`

Latitude and longitude were missing or could not be parsed as
numbers.

**Example.** `[Рє»SN-420.6] the geo URI is not in the correct format because latitude and longitude coordinates were expected`

**Resolution.** Provide two comma-separated decimal numbers
immediately after `geo:`.

### `SN-420.7` Рђћ `BadUncertainty`

The value of a `u=` (uncertainty) parameter was not a valid number.

**Example.** `[Рє»SN-420.7] the geo URI is not in the correct format because the `uncertainty` parameter vas not a valid number`

**Resolution.** Provide a non-negative decimal value for `u=`.

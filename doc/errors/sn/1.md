# `SN-sn/1` — `McpError`

Raised when an MCP (Model Context Protocol) operation fails.

## Cause

Synesthesia's MCP runtime dispatches incoming requests to
user-defined tools, prompts, and resources. When dispatch or
parameter binding fails, this error is raised. The constructor
takes a `reason: Reason`; the rendered message is `the MCP
operation failed because $reason`.

## Variants

### `SN-sn/1.1` — `UnknownMethod`

The method name in the request did not match any registered tool
or prompt — the wildcard fallback in the generated dispatcher.

**Example.** `[↯SN-sn/1.1] the MCP operation failed because the method name does not match any registered tool or prompt`

**Resolution.** Verify the method name matches one declared on the
server interface; check for typos or case mismatch.

### `SN-sn/1.2` — `UnknownResource`

The URI in a resource request did not match any of the URI
patterns declared by the server interface.

**Example.** `[↯SN-sn/1.2] the MCP operation failed because the URI does not match any registered resource`

**Resolution.** Confirm the URI matches one of the patterns
declared with `@resource(...)` on the interface.

### `SN-sn/1.3` — `MissingParameter`

A required parameter for a tool method was not present in the
input map of the request.

**Example.** `[↯SN-sn/1.3] the MCP operation failed because a required parameter was not present in the input`

**Resolution.** Send all required parameters in the JSON-RPC
arguments object, or mark the parameter as optional in the server
interface.

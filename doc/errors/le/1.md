# `SN-le/1` — `QueryError`

Raised when Legerdemain cannot read a query parameter.

## Cause

Legerdemain decodes named query parameters into typed values. The
constructor takes a `reason: Reason`; the rendered message is
`the query parameter could not be read because $reason`.

## Variants

### `SN-le/1.1` — `Missing`

The parameter was not present in the query string. (When the
target type has a `Default` instance, the default is used in
place of the missing value, but the error is still raised so a
caller's `Tactic` can observe it.)

**Example.** `[↯SN-le/1.1] the query parameter could not be read because the parameter was not present in the query string`

**Resolution.** Confirm the parameter is included in the query
string; supply a `Default[T]` instance to fall back when it is
absent.

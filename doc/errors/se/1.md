# `SN-se/1` — `PathError`

Raised when a path operation cannot proceed because the path is in
some way invalid.

## Cause

Serpentine reports `PathError` for structural problems with paths —
operating on the root, mismatched roots between paths, malformed
roots, or invalid name components. The message is `the path $path
was invalid because $reason` (the `path` prefix is omitted when not
known).

## Variants

### `SN-se/1.1` — `RootParent`

`parent` was called on a root path, which has no parent.

**Example.** `[↯SN-se/1.1] the path /  was invalid because the root has no parent`

**Resolution.** Check whether the path is a root before requesting
its parent, or use an `Optional`-returning accessor.

### `SN-se/1.2` — `InvalidRoot`

The root component itself is malformed (for example a Windows path
that does not start with a drive letter or UNC root).

**Example.** `[↯SN-se/1.2] the path was invalid because the root is not valid`

**Resolution.** Construct paths via the typed `Path` API rather than
parsing arbitrary text, or pass a syntactically valid root.

### `SN-se/1.3` — `DifferentRoots`

A relativisation or join was attempted between two paths that have
incompatible roots.

**Example.** `[↯SN-se/1.3] the path was invalid because it does not have the same root as the source`

**Resolution.** Ensure both paths share the same root before
relativising; otherwise, work in absolute paths throughout.

### `SN-se/1.4` — `InvalidName`

A path component (name) is invalid for the target filesystem — for
example contains illegal characters or is reserved.

**Example.** `[↯SN-se/1.4] the path was invalid because the name is not valid`

**Resolution.** Sanitise component names against the target
platform's rules before composing them into a path.

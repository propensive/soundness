# `SN-hi/2` — Encoding not available

Raised at compile time by Hieroglyph's `enc"…"` macro when the
named encoding is not supported by the JVM.

## Cause

The macro looks up the encoding via `Encoding.unapply`. If the JVM
does not recognise the name, the macro halts.

## Example message

```
[↯SN-hi/2] the encoding XYZ-9 was not available
```

## Resolution

Use a JVM-supported encoding name. Standard encodings like
`UTF-8`, `ISO-8859-1`, and `US-ASCII` are always present.

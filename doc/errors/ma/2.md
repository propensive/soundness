# `SN-ma/2` — Unsupported lambda

Raised at compile time by Mandible's `disassemble` macro when the
lambda expression's AST shape is not recognised.

## Cause

The macro pattern-matches on lambda expressions to extract their
underlying classfile entry. Some lambda forms — typically those
using anonymous inner classes or unusual desugarings — are not
recognised.

## Example message

```
[↯SN-ma/2] this type of lambda is not supported
```

## Resolution

Rewrite the lambda in a simpler form (e.g. `(x: Int) => x + 1`).

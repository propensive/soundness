# `SN-fu/1` — `EscapeError`

Raised by Fulminate when a string-interpolation escape sequence is
not valid.

## Cause

The `m"…"` and similar interpolators support escape sequences
(`\n`, `\t`, `\u…`, etc.). When an escape is malformed, the
interpolator throws this error with a message describing the
specific issue.

## Example message

```
[↯SN-fu/1] the escape sequence \q is not recognized
```

## Resolution

Use a documented escape sequence; for a literal backslash, write
`\\`.

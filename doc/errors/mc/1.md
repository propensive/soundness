# `SN-mc/1` — Companion has no candidate `apply` methods

Raised at compile time by Mercator's `identity` macro when the
companion object of the target type has no usable `apply` methods.

## Cause

Mercator derives `Identity` instances by invoking a companion's
`apply`. If none exist, derivation cannot proceed.

## Example message

```
[↯SN-mc/1] the companion object Foo has no candidate apply methods
```

## Resolution

Define an `apply` on the companion object that constructs a value
of the type, or supply an `Identity` instance manually.

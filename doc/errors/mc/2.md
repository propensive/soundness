# `SN-mc/2` — Companion has multiple candidate `apply` methods

Raised at compile time by Mercator's `identity` macro when the
companion has more than one applicable `apply` method.

## Cause

Mercator cannot pick one over another automatically; ambiguity
must be resolved by the user.

## Example message

```
[↯SN-mc/2] the companion object Foo has more than one candidate apply method
```

## Resolution

Remove or rename the extra `apply` overloads, or supply an
`Identity` instance manually.

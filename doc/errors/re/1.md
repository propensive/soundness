# `SN-re/1` — `SemverError`

Raised when a value is not a valid semantic version.

## Cause

Revolution validates input strings against the SemVer 2.0
grammar: `MAJOR.MINOR.PATCH[-prerelease][+build]`. The constructor
records the offending `version` and a `reason: Reason`; the
rendered message is `$version is not a valid semantic version
because $reason`.

## Variants

### `SN-re/1.1` — `BadFormat`

The string did not match the SemVer grammar at all (e.g. fewer
than three numeric components, or unrecognised separators).

**Example.** `[↯SN-re/1.1] 1.2 is not a valid semantic version because the string did not match the SemVer 2.0 grammar`

**Resolution.** Provide a fully-formed `MAJOR.MINOR.PATCH` string,
optionally followed by `-prerelease` and `+build`.

### `SN-re/1.2` — `EmptyIdentifier`

The string contained a `-` or `+` separator with nothing after it
— an empty prerelease or build identifier.

**Example.** `[↯SN-re/1.2] 1.2.3- is not a valid semantic version because a prerelease or build identifier was empty`

**Resolution.** Either remove the trailing separator or supply a
non-empty identifier after it.

### `SN-re/1.3` — `LeadingZero`

A numeric identifier (one of `MAJOR`, `MINOR`, `PATCH`, or a
numeric prerelease segment) had a leading zero, which SemVer
forbids.

**Example.** `[↯SN-re/1.3] 01.2.3 is not a valid semantic version because a numeric identifier had a leading zero`

**Resolution.** Strip leading zeros from numeric identifiers.

### `SN-re/1.4` — `InvalidCharacter`

A prerelease or build identifier contained a character outside
the allowed set `[0-9A-Za-z-]`.

**Example.** `[↯SN-re/1.4] 1.2.3-alpha! is not a valid semantic version because an identifier contained a character outside [0-9A-Za-z-]`

**Resolution.** Remove or escape the invalid character.
